# Task: Deep Research Button + Document Attachments

## Objective
Duas melhorias independentes no chat: (1) remover o Deep Research do grupo de flags junto ao webSearch e reposicioná-lo como botão de toggle ao lado do ícone de Reasoning, com o mesmo comportamento visual (verde quando ativo); (2) ampliar o suporte a anexos no chat para além de imagens, aceitando também documentos (txt, md, docx, PDF); (3) trocar o ícone de anexo de imagem para clipe (Paperclip) já que agora aceita múltiplos tipos.

## Context
- Chat UI: componente(s) responsáveis pela barra inferior do chat (área de input, flags, toggles de reasoning/webSearch/deepResearch)
- Attachment: lógica atual aceita apenas imagens; precisa ser estendida para documentos
- Stack: React + Tauri (Rust backend); modelos via API (provavelmente OpenAI/Claude com suporte a file/document input)

## Requirements
- [R1] Remover o toggle de Deep Research da área de flags (onde está junto ao webSearch)
- [R2] Adicionar botão de Deep Research ao lado do botão de Reasoning na barra do chat
- [R3] Comportamento do novo botão: clique alterna on/off; quando ativo fica esverdeado (mesmo padrão visual do Reasoning)
- [R4] Ícone: SVGs customizados `deep-gray.svg` / `deep-green.svg` em `src/assets/icons/`, com texto "Deep Research" ao lado
- [R5] Suporte a anexos de documentos: txt, md, PDF — além das imagens já suportadas (DOCX sem suporte — sem crate disponível)
- [R6] Texto extraído dos documentos enviado ao modelo como text blocks (compatível com qualquer modelo via OpenRouter)
- [R7] Trocar ícone do botão de anexo de imagem para `Paperclip` (clipe genérico)
- [R8] Persistência de documentos entre sessões via filesystem local (Tauri): texto extraído salvo em arquivo local, path referenciado no ChatMessage; documento disponível ao modelo em toda a conversa inclusive após reinício do app
- [R9] Limpeza automática dos arquivos locais ao deletar a conversa

## Constraints
- Manter o comportamento atual do webSearch flag inalterado
- Manter o comportamento atual do Reasoning inalterado
- Não quebrar o fluxo de envio de imagens existente
- Seguir o padrão visual existente (mesma classe CSS / Tailwind usada pelo botão de Reasoning)
- Ícone do Deep Research definido via SVG customizado (constante isolada para fácil troca)
- Conteúdo dos documentos NÃO vai no localStorage — apenas o path do arquivo é armazenado
- Arquivos de documentos salvos em subdiretório `chat-docs/` do app data dir do Tauri

## Edge Cases
- Usuário ativa Deep Research e webSearch simultaneamente — ambos devem funcionar de forma independente
- Arquivo de documento muito grande (limite: 5MB) — rejeitar com mensagem clara
- PDF escaneado sem texto extraível — mensagem de erro clara ao usuário
- Path de documento inválido ou arquivo deletado externamente — incluir texto de fallback na mensagem sem quebrar o envio
- Usuário tenta anexar tipo não suportado — mostrar feedback claro
- Deleção de conversa deve limpar todos os arquivos de documentos associados
