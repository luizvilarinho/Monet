# Plano (Revisado)

## Escopo

Adicionar suporte a **Watched Folders** na Knowledge Base do Monet. O usuário poderá cadastrar diretórios locais; o app fará embedding automático dos arquivos contidos neles, detectando novos, modificados e removidos. O scan ocorre no startup e via botão manual.

Incorpora os achados do Addendum do Explorer:
- Arquivos de pasta **não são copiados** para `documents_dir` — o `original_path` aponta diretamente para o disco do usuário.
- A coluna `is_external` na migration v10 protege `documents_delete` de apagar arquivos que pertencem ao usuário.

Nenhuma breaking change nas funcionalidades existentes de arquivos individuais.

### Etapa 3 — Proteger `documents_delete` contra deleção de arquivos externos

- **Status:** PENDENTE
- **Descrição:** Na função `documents_delete` em `documents.rs` (trecho das linhas 697–735), o SELECT inicial obtém apenas `original_path`. Alterar para incluir `is_external`:

  **Query atual:**
  ```rust
  let original_path: Option<String> = with_doc_db(&app, &state, |conn| {
      match conn.query_row(
          "SELECT original_path FROM documents WHERE id = ?1",
          ...
      )
  ```

  **Nova query:**
  ```rust
  let (original_path, is_external): (Option<String>, bool) = with_doc_db(&app, &state, |conn| {
      match conn.query_row(
          "SELECT original_path, is_external FROM documents WHERE id = ?1",
          params![document_id],
          |r| Ok((
              r.get::<_, Option<String>>(0)?,
              r.get::<_, bool>(1)?,
          )),
      ) {
          Ok(v) => Ok(v),
          Err(rusqlite::Error::QueryReturnedNoRows) => Ok((None, false)),
          Err(e) => Err(format!("failed to query document: {}", e)),
      }
  })?;
  ```

  **Trecho final** (substituir o bloco `if let Some(path) = original_path`):
  ```rust
  if let Some(path) = original_path {
      if !is_external {
          let _ = fs::remove_file(PathBuf::from(path));
      }
  }
  ```

  Nenhuma outra alteração nesta função. Arquivos individuais existentes continuam sendo apagados do disco (comportamento preservado, pois `is_external=0` por padrão).

### Etapa 7 — Lógica de scan: `scan_watched_folder_async`

- **Status:** PENDENTE
- **Descrição:** Implementar em `documents.rs` a função assíncrona interna `scan_watched_folder_async(app: AppHandle, folder_id: String)`. Fluxo:

  **7a. Ler pasta no banco:**
  - `SELECT original_path FROM documents WHERE id = ?1 AND type = 'folder'`.
  - Se não encontrar: retornar silenciosamente.

  **7b. Verificar existência no disco:**
  - Se `fs::metadata(folder_path).is_err()` ou não for diretório: atualizar `status='error'`, `error_message='Folder not found on disk'`, emitir evento `documents://status` e retornar.

  **7c. Coletar arquivos em disco (recursivo):**
  - Usar `std::fs::read_dir` recursivamente. Para cada entrada de arquivo (não diretório):
    - Pular se `is_dir()` (recursão continua, não adiciona à lista).
    - Pular symlinks: `metadata.file_type().is_symlink()`.
    - Pular se `detect_mime()` retornar `None` (tipo não suportado).
    - Em caso de `PermissionDenied`: logar via `eprintln!` e continuar (não abortar).
    - Ler `last_modified_ms` via `fs::metadata(path).modified()` e converter para milissegundos.
  - Resultado: `Vec<(PathBuf, i64)>` — `(absolute_path, last_modified_ms)`.

  **7d. Buscar filhos já indexados no banco:**
  - `SELECT id, original_path, last_modified_ms FROM documents WHERE parent_folder_id = ?1 AND type = 'file'`.
  - Construir mapa: `HashMap<String, (String, Option<i64>)>` de `original_path → (id, last_modified_ms)`.

  **7e. Detectar mudanças e agir:**
  - Para cada arquivo em disco (pelo `absolute_path.to_string_lossy()`):
    - **Não existe no mapa (arquivo novo):**
      - Inserir em `documents` com:
        - `type = 'file'`
        - `parent_folder_id = folder_id`
        - `status = 'indexing'`
        - `mime = detect_mime(path)` (string)
        - `size = metadata.len()`
        - `original_path = absolute_path` (caminho real no disco — sem cópia)
        - `last_modified_ms = valor lido`
        - `is_external = 1`
      - Chamar `run_indexing(app.clone(), new_id, name, absolute_path, mime)` em `tauri::async_runtime::spawn`.
    - **Existe e `last_modified_ms` é diferente (arquivo modificado):**
      - `UPDATE documents SET last_modified_ms=?, status='indexing', updated_at=? WHERE id=?`.
      - Chamar `delete_chunks_for_document` para o id existente.
      - Chamar `run_indexing(app.clone(), existing_id, name, absolute_path, mime)` em spawn.
    - **Existe e `last_modified_ms` é igual:** pular.

  **7f. Deletar arquivos removidos do disco:**
  - Para cada entrada do mapa que NÃO aparece na lista do disco: chamar `delete_folder_child(&app, id)`.

  **7g. Atualizar status do folder:**
  - Se não houve erro catastrófico: `UPDATE documents SET status='available', updated_at=? WHERE id=?`.
  - Emitir `documents://status` com `status='available'` para o `folder_id`.

  Adicionar também o comando público `documents_scan_watched_folder(app, state, folder_id: String)` que chama `scan_watched_folder_async` em spawn e retorna `Ok(())` imediatamente. Registrar no `invoke_handler`.
- **Arquivos:** `src-tauri/src/documents.rs`, `src-tauri/src/lib.rs`

## Notas de implementação

### Arquivos externos vs. internos

| Propriedade | Arquivo individual (existente) | Folder (novo) | Arquivo filho de pasta (novo) |
|---|---|---|---|
| `type` | `'file'` | `'folder'` | `'file'` |
| `parent_folder_id` | NULL | NULL | ID do folder pai |
| `is_external` | `0` | `1` | `1` |
| `original_path` | Aponta para `documents_dir` (cópia) | Aponta para disco do usuário | Aponta para disco do usuário |
| Cópia ao inserir? | SIM (linha 607 de `documents_upload_global`) | NÃO (apenas registra) | NÃO (apenas registra via scan) |
| Ao reindexar | Lê de `documents_dir` | N/A | Lê do disco do usuário |
| Ao deletar | Remove arquivo físico + banco | Remove banco (sem arquivo físico) | Remove banco (sem arquivo físico) |
| Pipeline `run_indexing` | Agnóstico | N/A | Agnóstico — funciona igual |

### `documents_upload_global` não muda

O INSERT existente não especifica `type`, `parent_folder_id`, `last_modified_ms` nem `is_external`. Os valores default (`'file'`, NULL, NULL, `0`) preenchem corretamente. Nenhuma alteração nessa função.

### Atualização de status de folder

O evento `documents://status` usa `document_id`, que para folders é o `folder_id`. O frontend já escuta e atualiza o item pelo id — funciona sem mudança no listener.

### Pasta removida do disco

Detectada no passo 7b do scan: marca `status='error'`, emite evento. O usuário pode remover manualmente via botão `remove` na UI ou recriar a pasta no disco e fazer rescan.

### `indeterminate` no DocumentsModal

Usar `ref` em `<input type="checkbox">` e setar `ref.current.indeterminate = true` via `useEffect` — o atributo não é suportado como prop JSX no React.
