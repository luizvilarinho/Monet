# Explorer Report — Web Search via Tool Use

## 1. Streaming SSE em `useChat.ts`

**Arquivo:** `src/hooks/useChat.ts` | **Linhas críticas:** 967-1029, 1263-1268

### Como a stream é consumida
- `onOpenRouterChunk()` (967-977): Acumula texto em `m.content` concatenando chunks
- `onOpenRouterReasoning()` (978-988): Acumula thinking em `m.thinking`
- `onOpenRouterDone()` (989-1006): Finaliza stream, calcula tokens/sec, reseta `deepResearchPhase` para null
- `onOpenRouterError()` (1007-1021): Trata erro e limpa estado
- Chunk chega via evento Tauri `openrouter://chunk` → handler recebe `{requestId, text}` → `updateMessages()` concatena `m.content + text`
- Stream iniciada em `send()` via `startOpenRouterStreamMessages({requestId, model, messages, thinking})` (linha 1263-1268)

### Onde interceptar `tool_calls`
- Ponto de interceptação no Rust: `lib.rs:799-830` — loop de processamento SSE, atualmente processa `delta.get("content")` e `delta.get("reasoning")`
- Novo ponto: verificar `delta.get("tool_calls")` (array com `{id, type, function: {name, arguments}}`)
- Formato OpenRouter (segue OpenAI):
  ```json
  {
    "choices": [{
      "delta": {
        "tool_calls": [{"id": "call_abc", "type": "function", "function": {"name": "web_search", "arguments": "{\"query\":\"...\"}"}}]
      },
      "finish_reason": "tool_calls"
    }]
  }
  ```

### Cancelamento da stream
- Localização: `useChat.ts:1290-1304` — função `cancel()`
- Invoca `cancelOpenRouterStream(requestId)` → oneshot channel em Rust
- Limpeza: `activeStreamRef.current = null`, `isStreaming = false`, `deepResearchPhase = null`

## 7. Viabilidade

**Nível:** MÉDIA — viável e bem-mapeada.

Favoráveis:
- Arquitetura SSE existente permite adição de novo tipo de delta sem refatores maiores
- Padrão `deepResearchPhase` replicável diretamente para `webSearchActive`
- Estilos CSS já existem (reuse)
- OpenRouter segue padrão OpenAI

Pontos de atenção:
- Validar estrutura exata de `supported_parameters` na resposta da API OpenRouter
- Tool calls chegam como delta discreto, não em chunks incrementais — stream para ao detectar `finish_reason: "tool_calls"`
- Argumentos chegam como string JSON — necessário parse com fallback

**Recomendação:** Proceder ao plannerAgent.
