### 5.3 Lógica de Scan

```
Para cada arquivo recursivo em folder_path:
  1. Validar tipo (pdf, txt, md)
  2. Ler last_modified_ms
  3. Buscar documento existente
  4. Se não existe: criar novo (type='file', parent_folder_id=folderId)
  5. Se existe e mudou: reindex
  6. Se existe e igual: pular
  
DELETE documentos órfãos (cascata de chunks)
UPDATE folder status='available'
```

## Addendum — File Copy vs Path Reference

### Questão Levantada

No contexto de Watched Folders, existe uma divergência crítica: o sistema atual (`documents_upload_global`) **copia o arquivo para `documents_dir` do app**. Para Watched Folders esse comportamento não é apropriado — copiar centenas de arquivos para dentro do app é ineficiente. Arquivos de pasta devem ser **lidos no lugar**, via `original_path` no disco do usuário.

Essa re-exploração mapeia onde a cópia acontece, como o pipeline lê o arquivo, e como distinguir entre dois modos no banco de dados.

---

### 1. Onde a Cópia Acontece

**Arquivo:** `src-tauri/src/documents.rs`  
**Função:** `documents_upload_global()` (linhas 567-631)  
**Linha crítica:** **607**

Código-chave (linhas 605-607):
```rust
let dest = documents_dir(&app)?.join(format!("{}.{}", id, ext));
fs::copy(&src, &dest).map_err(|e| format!("failed to copy file: {}", e))?;
```

**Fluxo:**
1. `documents_upload_global(source_path)` recebe o caminho do arquivo original
2. Linha 605: gera um `dest` dentro de `documents_dir` com nome `{uuid}.{ext}`
3. Linha 607: **copia o arquivo** do `source_path` para `dest`
4. Linhas 612-619: insere na tabela com `original_path = dest` (o caminho **copiado**, não o original)

**Resultado:** O banco guarda o caminho do arquivo **dentro do app**, e o arquivo original é abandonado.

---

### 2. Como o Pipeline de Indexação Acessa o Arquivo

**Arquivo:** `src-tauri/src/documents.rs`  
**Função:** `run_indexing()` (linhas 498-564)

Linha 517 — entrada do arquivo:
```rust
let text = match read_document_text(path, mime).await {
```

E duas origens desse `path`:

**Origem 1 — `documents_upload_global` (linha 627):**
```rust
run_indexing(app_clone, id_clone, name_clone, dest, mime_str).await;
//                                             ^^^^
// Passa 'dest' (arquivo COPIADO para documents_dir)
```

**Origem 2 — `documents_reindex` (linha 686):**
```rust
let path = PathBuf::from(original_path);  // Lê do banco
run_indexing(app_clone, id_clone, name, path, mime).await;
//                                     ^^^^
// Passa 'original_path' do banco (que aponta para documents_dir ou disco)
```

**Conclusão:** O pipeline **nunca copia** — ele lê do caminho recebido. O problema é que `original_path` no banco aponta para `documents_dir` (cópia), não para o original do usuário.

---

### 3. Fluxo de Deleção e Risco Crítico

**Arquivo:** `src-tauri/src/documents.rs`  
**Função:** `documents_delete()` (linhas 697-735)

Trecho crítico (linhas 730-732):
```rust
if let Some(path) = original_path {
    let _ = fs::remove_file(PathBuf::from(path));  // <-- SEM PROTEÇÃO
}
```

**Risco para Watched Folders:**
- Para **"file individual"**: `original_path` aponta para `documents_dir` → apagar é seguro
- Para **"folder file"**: `original_path` apontaria para disco do usuário → **apagar deletaria o arquivo original!**

Exemplo de desastre:
```
Usuário registra: /home/user/projects/reports/q1.pdf
App insere: is_external=1, original_path=/home/user/projects/reports/q1.pdf
Usuário clica "delete" na KB
documents_delete executa: fs::remove_file("/home/user/projects/reports/q1.pdf")
RESULTADO: Arquivo original deletado do disco!
```

---

### 4. Solução: Campo `is_external` na Tabela

**Migration v10 — Adicionar:**
```sql
ALTER TABLE documents ADD COLUMN is_external BOOLEAN NOT NULL DEFAULT 0;
-- 0: arquivo copiado (documents_dir) — apagar é seguro
-- 1: arquivo referenciado (disco do usuário) — NÃO APAGAR
```

**Índice recomendado:**
```sql
CREATE INDEX IF NOT EXISTS idx_documents_is_external ON documents(is_external);
```

```rust
pub async fn documents_delete(...) -> Result<(), String> {
    let (original_path, is_external): (Option<String>, bool) = 
        with_doc_db(&app, &state, |conn| {
            conn.query_row(
                "SELECT original_path, is_external FROM documents WHERE id = ?1",
                params![document_id],
                |r| Ok((Some(r.get::<_, String>(0)?), r.get::<_, bool>(1)?)),
            )
        })?;
    
    delete_chunks_for_document(&app, &document_id)?;
    with_doc_db(&app, &state, |conn| {
        // ... delete from dbs ...
    })?;
    
    // PROTEÇÃO: só apagar arquivo se é interno
    if let Some(path) = original_path {
        if !is_external {  // <-- GARANTIA: protege arquivos do usuário
            let _ = fs::remove_file(PathBuf::from(path));
        }
    }
    Ok(())
}
```
