## Feature 2 — Suporte a Documentos em Anexos

**Pipeline atual de imagens** (`ChatPanel.tsx`):
- `handlePickImage()` linhas 202-221
- Input: `accept='image/*'`
- Leitura como base64 DataURL via `FileReader.readAsDataURL()`
- Limite: 5MB
- Preview: linhas 338-350
- Clipboard paste: linhas 388-399

**Construção de mensagem** (`useChat.ts` linhas 1266-1271):
```
{ type: 'image_url', image_url: { url: imageDataUrl } }
```
Campo na mensagem: `imageDataUrl?: string` (linha 102)

**Backend** (`src-tauri/src/lib.rs`):
- Linha 682-685: mensagens serializadas e enviadas diretamente à API OpenRouter
- Sem pré-processamento — content é `serde_json::Value` flexível

**API OpenRouter/Claude — suporte a documentos**:
- ✅ Imagens: `{ type: 'image_url', image_url: { url: string } }`
- ✅ Documentos (Claude 3.5+): `{ type: 'document', source: { type: 'base64', media_type: string, data: string } }`
- MIME types aceitos: `application/pdf`, `text/plain`, `text/markdown`, `.docx`

**Mudanças necessárias**:
1. `src/lib/openrouter.ts`: adicionar `DocumentContentBlock` aos tipos
2. `ChatPanel.tsx`: ampliar `accept` e criar lógica de leitura por tipo
3. `useChat.ts`: atualizar `send()` para receber documentos, construir content blocks
4. Validação: limites de tamanho, whitelist de MIME types

**Riscos**:
- HIGH: compatibilidade do modelo (modelos antigos não suportam `document` block)
- MEDIUM: DOCX corrompido; interação com Deep Research ativo
- LOW: complexidade de implementação
