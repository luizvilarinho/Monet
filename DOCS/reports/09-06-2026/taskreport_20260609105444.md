# Task: Tray + Mini Chat Assistant (Ctrl+M)

## Objective
Make Monet run from the Windows system tray and add a global quick-assistant: closing the main window (X) hides the app to the tray instead of quitting, and a global `Ctrl+M` shortcut summons a small separate "assistant" window for instant questions. Assistant conversations are persisted under a default chat folder named `assistant`, sharing the existing chat history so they are also visible in the main window.

## Context
- Stack: Tauri v2 + React + TypeScript. Single window `main` today (`src-tauri/tauri.conf.json`).
- Backend setup: `src-tauri/src/lib.rs` `.setup` closure (~line 1151) and plugin chain (~1174). Currently no tray and no global-shortcut plugin.
- `Cargo.toml`: `tauri = { version = "2", features = [] }`; has `tauri-plugin-process`, `tauri-plugin-updater`, `tauri-plugin-opener`, `tauri-plugin-sql`. No `tray-icon` feature, no `tauri-plugin-global-shortcut`.
- Capabilities: `src-tauri/capabilities/default.json` targets `windows: ["main"]`.
- Chat: `src/hooks/useChat.ts` persists everything in `localStorage` — conversations (`CONVERSATIONS_KEY`), chat folders (`FOLDERS_KEY`, ~line 288), active id, model. The chat already supports organizing conversations into folders. `localStorage` is shared across windows of the same origin, so a second window reuses the same history automatically.
- Chat UI: `src/components/ChatPanel/ChatPanel.tsx` and related. Main app shell + existing in-app keyboard shortcuts in `src/App.tsx` (~line 282: `Ctrl+Space`, `Ctrl+P`).
- Tray icon asset available: `src-tauri/icons/icon.ico`.

## Requirements

### 1 — Close-to-tray (X always hides)
- Add a Windows system tray icon with a menu exposing at least: **Open** (show + focus main window) and **Quit** (fully exit the app).
- Intercept the main window close button (`WindowEvent::CloseRequested`): always `prevent_close()` and hide the window instead of quitting. The X never terminates the app.
- The only way to fully quit is the tray menu **Quit** item.
- Clicking the tray icon (left click) and the **Open** menu item both show + focus the main window.

### 2 — Global `Ctrl+M` shortcut → assistant window
- Register a global OS-level shortcut `Ctrl+M` (works even when the app is hidden in the tray / not focused), via `tauri-plugin-global-shortcut`.
- `Ctrl+M` toggles a separate small `assistant` window: if hidden, show + focus it; if already visible and focused, hide it.

### 3 — Separate mini "assistant" window
- Declare a second window labelled `assistant` in `tauri.conf.json`: small size, `visible: false` at startup, `alwaysOnTop: true`, `skipTaskbar: true`. Decorations are the implementer's call (favor a clean, compact look; a minimal title bar or none).
- The assistant window renders a **compact** version of the chat (reuse `ChatPanel` / `useChat` logic — do not fork the chat engine). It must let the user type a question, see the streamed answer, and dismiss the window.
- Detect the window by its label (e.g. via `getCurrentWindow().label === 'assistant'`) to switch the React app into compact-assistant mode instead of rendering the full app shell.

### 4 — Persist assistant questions in an `assistant` folder
- Ensure a chat folder named `assistant` exists (create on first use) using the existing chat-folders mechanism in `useChat.ts`.
- Conversations started from the assistant window are created/stored inside that `assistant` folder.
- Because `localStorage` is shared, these conversations must also appear under the `assistant` folder in the main window's chat sidebar — no separate storage path.

## Constraints
- **Authorized config/dependency changes (explicitly in scope for this task):** the codeAgent MAY edit `Cargo.toml` (enable `tray-icon` feature on the `tauri` crate; add `tauri-plugin-global-shortcut = "2"`), `src-tauri/tauri.conf.json` (add the `assistant` window), and `src-tauri/capabilities/` (permissions for tray, global-shortcut, and window show/hide; add the `assistant` window to the relevant capability). These are normally on the codeAgent "do not touch" list — they are pre-approved here. Do NOT touch anything else on that list (no DB migrations/schema changes, no unrelated config).
- **No DB schema changes.** Chat (including the `assistant` folder) lives entirely in `localStorage`; do not add tables or migrations.
- Reuse the existing chat engine/hooks (`useChat`, `ChatPanel`) for the assistant window — do not reimplement chat or duplicate the streaming/persistence logic.
- Minimal diff. Match existing code patterns (the existing keydown handling, the existing plugin registration order, the existing window setup in `.setup`).
- CSP / network: the assistant window uses the same OpenRouter flow as the main chat; do not loosen `tauri.conf.json` CSP beyond what already exists.
- App is pre-launch with no active users — breaking changes to local state are acceptable, but avoid corrupting existing chat history (the `assistant` folder must coexist with current conversations/folders).

## Edge Cases
- `Ctrl+M` pressed while the assistant window is already open and focused → hide it (toggle behavior), don't spawn duplicates.
- `Ctrl+M` pressed while the app is fully hidden in the tray → assistant window still appears (global shortcut must be OS-level, not an in-app JS listener).
- Main window hidden in tray + user triggers assistant → assistant works independently; opening the assistant must not force-show the main window.
- Quit from tray must close ALL windows (assistant + main) and terminate the process, not just hide.
- Global-shortcut registration failure (e.g. `Ctrl+M` already taken by another app) must not crash startup — fail gracefully (log and continue).
- The `assistant` chat folder must be created only once (idempotent) even across many assistant sessions.
- Shared `localStorage`: writes from the assistant window and the main window target the same keys — ensure the assistant uses the same persistence helpers so the main window reflects new assistant conversations without conflict.
- Single-instance is NOT in scope unless trivially required; if launching a second app instance becomes an issue, flag it rather than adding a single-instance plugin unprompted.
