# Plano

## Escopo
Três melhorias na funcionalidade de agenda (Calendar notebook):
1. Suprimir o painel de Related Content quando a nota ativa pertence ao calendar notebook
2. Habilitar o NotePicker (embed de notas) no Editor para notas de agenda, com acesso a todas as notas do app
3. Habilitar RAG nas notas de agenda via campo `rag_doc_ids` por nota (override do sistema atual baseado em notebook)

- `PENDENTE` — etapa ainda não iniciada
- `FEITO` — etapa concluída sem ressalvas
- `REVISAR` — etapa executada, mas ainda depende de ajuste

### Etapa 6 — UI para configurar `ragDocIds` numa nota de agenda
- **Status:** PENDENTE
- **Descrição:** No componente `Editor` (`src/components/Editor/Editor.tsx`), adicionar uma prop opcional `ragDocIds?: string[] | null` e uma prop `onRagDocIdsChange?: (ids: string[]) => void`. Quando a nota ativa for uma nota de agenda (`isCalendarNote` booleano passado como prop, ou derivado de `notebookId === calendarNotebookId`), renderizar um pequeno painel abaixo do título (similar ao campo `date` já existente) com um seletor de documentos: lista os documentos disponíveis (via prop `availableDocuments?: Document[]`) com checkboxes, e ao marcar/desmarcar chama `onRagDocIdsChange`.

  Em `src/App.tsx`, passar as novas props ao `<Editor>` quando `activeNote.notebookId === calendarNotebookId`:
  - `ragDocIds={activeNote.ragDocIds}`
  - `onRagDocIdsChange={(ids) => updateActive({ ragDocIds: ids })}`
  - `availableDocuments={documents}` (array de documentos já disponível no estado)
  - `isCalendarNote={activeNote.notebookId === calendarNotebookId}`

  **Nota:** manter o painel colapsado por padrão (tipo accordion) para não poluir o editor. Exibir label "Knowledge Base" com o número de documentos selecionados, se houver.
- **Arquivos:** `src/components/Editor/Editor.tsx`, `src/App.tsx`
- **Depende de:** Etapa 4, Etapa 5
