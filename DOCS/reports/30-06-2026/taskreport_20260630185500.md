# Task: Comando /week para notas de calendário

## Objective
Implementar o slash command `/week`, disponível somente em notas do notebook Calendar, que reúne o conteúdo das notas de calendário dos últimos 7 dias (âncora: data da nota atual) e usa esse conjunto como contexto para o modelo responder perguntas livres sobre a semana do usuário (ex.: "quais foram minhas atividades dessa semana?").

## Context
- Sistema de slash commands: `src/lib/commands.ts` (registro `COMMANDS`), `src/components/Editor/commandParser.ts` (parsing/autocomplete), `src/components/Editor/CommandExtension.ts` + `CommandAutocomplete.tsx` (UI no editor).
- Orquestração de execução de comando: `src/App.tsx`, função `handleCommand` (constrói contexto e chama o modelo), `SYSTEM_PROMPT` e `buildUserMessage` (App.tsx).
- Notebook Calendar: `CALENDAR_NOTEBOOK_ID = 'calendar'` (App.tsx); notas desse notebook são identificadas pelo título no formato `DD/MM/YYYY` (criadas via `handleCalendarDayClick`).
- Não existe hoje nenhum mecanismo de comando restrito por notebook, nem agregação de conteúdo de múltiplas notas como contexto — ambos são capabilities novas.

## Requirements
- Novo comando `/week` em `src/lib/commands.ts`, com `takesQuery: true` e `usesSearch: false` (não usa web search, somente contexto interno das notas).
- `/week` só deve estar disponível (aparecer no autocomplete e ser executável) quando a nota atual pertence ao notebook Calendar (`notebookId === CALENDAR_NOTEBOOK_ID`). Fora desse contexto, o comando não deve ser sugerido nem executado.
- Ao executar `/week`, montar o contexto buscando todas as notas do notebook Calendar cujo título (`DD/MM/YYYY`) caia nos últimos 7 dias, usando como âncora a data da nota atual (inclusive o próprio dia).
- Concatenar o conteúdo dessas notas (com rótulo de data por nota) como bloco de contexto adicional na mensagem enviada ao modelo, reaproveitando o padrão já usado por `buildUserMessage` / blocos de RAG / web search.
- Acrescentar regra de comportamento para `/week` no `SYSTEM_PROMPT`, orientando o modelo a responder com base nas notas da semana fornecidas.
- Resposta segue o fluxo de streaming/exibição já existente para os demais comandos (sem nova UI além do necessário para a restrição por notebook).

## Constraints
- Não criar pipeline de embeddings/RAG para notas — o contexto é concatenação direta do conteúdo bruto, sem chunking/sumarização (decisão já validada com o Coordenador: o usuário assume o custo de tokens).
- Não alterar o comportamento dos demais comandos existentes.
- Reutilizar os padrões já existentes em `App.tsx` (`buildUserMessage`, blocos de contexto) em vez de criar uma estrutura paralela.
- Projeto está em fase de pré-lançamento sem usuários ativos — breaking changes são aceitáveis se necessário, mas não é o objetivo desta task.

## Edge Cases
- Dias da janela de 7 dias sem nota de calendário correspondente: omitir silenciosamente, sem inventar conteúdo.
- Nota de calendário vazia (sem conteúdo): omitir ou marcar como "sem conteúdo", a critério da implementação, mas nunca quebrar a montagem do contexto.
- Usuário digita `/week` manualmente em uma nota fora do notebook Calendar: comando não deve ser reconhecido/executado (mesmo comportamento de um texto não-comando).
- A própria nota atual (onde `/week` foi digitado) deve ser incluída no cálculo da janela dos 7 dias, já que ela também é uma nota de calendário do período.
