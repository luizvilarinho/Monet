### Etapa 6 — Item 5: BUG — Spell-check no menu de contexto
- **Status:** PENDENTE
- **Descrição:** O handler `contextmenu` em `src/components/Editor/Editor.tsx` sempre chama `e.preventDefault()`, bloqueando sugestões nativas de spell-check.

  **Abordagem:** No handler do evento `contextmenu`, verificar se há seleção de texto ativa. Se não houver seleção (usuário clicou sem selecionar — típico de right-click em palavra com erro), não chamar `e.preventDefault()` para que o menu nativo apareça:
  ```ts
  const handler = (e: MouseEvent) => {
    const selection = window.getSelection()
    const hasSelection = selection && selection.toString().length > 0
    if (hasSelection) {
      e.preventDefault()
    }
    setContextMenuPos({ x: e.clientX, y: e.clientY })
  }
  ```
  **Nota:** Testar no Windows 11 com Tauri/WebView2 — comportamento pode variar. Se a abordagem não funcionar, alternativa é remover `e.preventDefault()` completamente. Registrar resultado no execution report.

- **Arquivos:** `src/components/Editor/Editor.tsx`
