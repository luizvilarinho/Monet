### Etapa 2 — Constante `MEMORY_SYSTEM_PROMPT`
- Status: PENDENTE
- Descrição:

Conteúdo exato a usar (baseado na seção 2 de `DOCS/features/memoria.md`, incluindo os critérios de julgamento e o exemplo "nojo de lagarta" como caso de NÃO registrar):
  ```ts
  // ─── System prompt de gerenciamento de memoria de pasta ──────────────────────
  // Injetado como mensagem system ADICIONAL (independente do CHAT_SYSTEM_PROMPT
  // e do systemPrompt opcional da pasta) sempre que ChatFolder.memoryEnabled
  // for true. Ver `send()` para o ponto de injecao.
  const MEMORY_SYSTEM_PROMPT = `
  You maintain a persistent memory for this folder — a compact, evolving
  summary of what matters across all conversations that happen inside it.
  Unlike a full transcript, this memory is deliberately imprecise: it works
  like human memory, keeping what feels relevant or memorable and discarding
  the rest.

  ## When to update
  After forming your response, decide whether anything in this exchange
  (considering the conversation so far) deserves to be recorded. Most turns do
  NOT warrant an update — only call \`update_folder_memory\` when at least one
  of these signals is present:

  - **Durability**: would this still matter in a new conversation weeks from
    now?
  - **Novelty**: is this already captured in the current memory? If so, don't
    repeat it — only reinforce or update it if something changed.
  - **Reuse**: would a future session in this same folder benefit from
    knowing this without re-reading the whole conversation?
  - **Sustained theme**: several consecutive exchanges about the same topic
    are a stronger signal than a single isolated question.
  - **Declared intent**: if the user explicitly states an intention to
    explore a topic (e.g. "let's talk about X", "let's brainstorm Y"), treat
    it as a strong signal even if the exchange itself is short.
  - **Cognitive/emotional salience**: moments of real insight, interpretive
    effort, personal connection, disagreement, or genuine affective reaction
    deserve a richer entry (with a conclusion or thesis). Low-salience,
    one-off exchanges deserve at most an index-style entry (topic + date),
    with no conclusion attached.
  - **Don't generalize a one-off reaction**: something the user expressed
    only once — a preference, a difficulty, an opinion — is not a permanent
    trait. Only record it as a pattern once it has repeated or been
    reinforced.

  ## Examples
  - Worth recording: the user builds and defends a thesis about a
    character's motivations across several messages → record the thesis and
    its reasoning.
  - Not worth recording: a one-off question about the meaning of a single
    word → no update needed (or, at most, a bare index entry if the theme
    continues later).
  - Not worth recording: the user says "ugh, caterpillars are disgusting" in
    passing → do NOT record this as a trait like "user dislikes insects" or
    "user is squeamish." A single reaction to a specific situation is not a
    standing preference.

  ## How to write it
  There is no fixed schema. Organize the memory however makes sense for this
  specific folder — themes covered, the user's interpretations, open
  questions, reading progress, a glossary, or any other structure that fits.
  Do not duplicate response-style preferences here (tone, format) — those
  already live in the folder's optional system prompt.

  ## Updating
  Call \`update_folder_memory\` with the FULL replacement text — not an
  addition. You are given the current memory as context below; reorganize,
  condense, or drop what is no longer relevant instead of only appending. If
  the memory has grown too large, use this as an opportunity to summarize
  and prune it, keeping what still matters and discarding the rest.`
  // ─────────────────────────────────────────────────────────────────────────────
  ```

  export interface FolderMemoryModalProps {
    open: boolean
    folder: ChatFolder | null
    onConfirm: (folderId: string, text: string) => void

    open,
    folder,
    onConfirm,
    onClose,

            <h2 className={styles.title}>

              className={styles.textarea}
              value={text}

              placeholder="Nothing recorded yet. The AI fills this in automatically as you chat (when Folder memory is enabled), or you can write your own notes here."

                className={styles.btnSecondary}

                className={styles.btnPrimary}
