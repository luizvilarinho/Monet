# Análise de Impacto — Foco automático no campo nome da nota ao criar nova nota

**Data:** 2026-05-20

---

## 1. Descrição do problema

Quando o usuário cria uma nova nota (clicando no botão "+" da sidebar ou no botão "New note" do `EmptyEditor`), o cursor não vai automaticamente para o campo de nome da nota. O usuário precisa clicar manualmente no campo "note title" para começar a digitar o nome, o que quebra o fluxo de criação.

**Objetivo:** ao criar uma nova nota, o cursor (foco do teclado) deve se mover automaticamente para o campo de título (`<input placeholder="note title">`) no componente `Editor`.

---

## 2. Como funciona a criação de notas hoje

O fluxo mapeado é:

```
Sidebar [+] ou EmptyEditor [New note]
  → App.handleCreateNote()
    → useNotes.create(activeNotebookId)
      → cria objeto Note com title: '', content: ''
      → salva no SQLite via storage.saveNote()
      → retorna a nota
    → setActiveId(note.id)
    → activeNote passa a ser a nova nota (não-nulo)
  → Editor monta (substitui EmptyEditor no DOM)
  → <input className={styles.title}> renderiza com value="" (sem foco)
```

O componente `Editor` (`src/components/Editor/Editor.tsx`) renderiza um `<input>` para o título na linha ~292:

```tsx
<input
  className={styles.title}
  value={title}
  onChange={(e) => onTitleChange(e.target.value)}
  placeholder="note title"
/>
```

Este input **não possui** atributo `autoFocus` nem `ref`, portanto nunca recebe foco automaticamente.

---

## 3. Análise de impacto

### 3.1 O que precisa ser alterado

Apenas **1 arquivo**: `src/components/Editor/Editor.tsx`

A alteração consiste em:
1. Adicionar um `useRef<HTMLInputElement>` para o input de título
2. Adicionar um `useEffect` que foca o input quando `title` está vazio (indicando nota recém-criada sem nome)

### 3.2 Quando o Editor monta/desmonta

O `Editor` só é renderizado quando `activeNote` é não-nulo. Quando não há nota ativa, o `EmptyEditor` é exibido. Portanto, ao criar uma nota:

- `EmptyEditor` desmonta
- `Editor` monta com `title=""` e `value=""`

Isso significa que o `useEffect` com `[]` ou condicionado a `title === ''` será executado exatamente no momento certo: quando o Editor acaba de montar por causa de uma nota nova.

### 3.3 Efeitos colaterais

| Cenário | Comportamento esperado |
|---|---|
| Criar nota nova (title="") | ✅ Input de título recebe foco |
| Selecionar nota existente com título | ❌ Input NÃO recebe foco (o usuário quer ver/editar o conteúdo) |
| Selecionar nota sem título (criada antes mas nunca nomeada) | ⚠️ Input recebe foco — aceitável, pois a nota ainda não tem nome |
| Limpar o título de uma nota existente | ⚠️ Input recebe foco se o useEffect reagir a `title` — evitar isso usando uma flag |

O caso de limpar o título é um edge case. Para evitá-lo, recomenda-se usar uma flag `isNewNote` como prop ou verificar se ambos `title === ''` e `value === ''` (nota sem conteúdo também é nova). Alternativamente, usar `autoFocus` puro (só dispara no mount inicial) resolve esse caso naturalmente.

### 3.4 Áreas não afetadas

- **Sidebar**: sem alterações — o botão "+" já chama `onCreate` corretamente
- **EmptyEditor**: sem alterações — o botão "New note" já chama `onCreate` corretamente
- **useNotes**: sem alterações — o hook já cria a nota com `title: ''`
- **App.tsx**: sem alterações necessárias — o fluxo `handleCreateNote` já funciona
- **Rust/Tauri**: sem alterações — mudança é puramente frontend
- **SQLite**: sem alterações — não afeta persistência

## 4. Dificuldade

**Baixa**

É uma alteração pontual em um único componente React, sem impacto em outros módulos, sem mudança de API, sem alteração no backend. Consiste em adicionar ~5 linhas de código (ref + useEffect + focus).

---

## 5. Recomendações

### Abordagem recomendada

Adicionar um `ref` no input de título do `Editor.tsx` e focá-lo somente quando a nota está vazia (title e content vazios), indicando que é uma nota recém-criada:

```tsx
const titleRef = useRef<HTMLInputElement>(null)

useEffect(() => {
  // Foca o título apenas em notas novas (title e content vazios)
  if (title === '' && value === '') {
    titleRef.current?.focus()
  }
}, []) // array vazio = só dispara no mount
```

Com `[]` como dependências, o efeito só roda uma vez, quando o Editor monta. Isso garante que:
- Nota nova recém-criada → foco no título ✅
- Navegar entre notas existentes → sem foco automático ✅
- Limpar título de nota existente → sem foco automático ✅

### Alternativa (ainda mais simples)

Usar `autoFocus` condicional diretamente no JSX:

```tsx
<input
  className={styles.title}
  value={title}
  onChange={(e) => onTitleChange(e.target.value)}
  placeholder="note title"
  autoFocus={title === ''}
/>
```

Porém, `autoFocus` só funciona no mount inicial. Como o Editor **não** desmonta/remonta ao trocar de nota (o React reutiliza o componente e só atualiza props), o `autoFocus` prop só teria efeito na primeira vez que o Editor aparece após o EmptyEditor. Isso é exatamente o comportamento desejado e é ainda mais simples que o useEffect.

### Recomendação final

Usar a abordagem com `useRef` + `useEffect` com dependências `[]` (vazio), pois é explícita e previsível. O `autoFocus` também funciona, mas depende do comportamento de reconciliação do React que pode ser menos óbvio para outros desenvolvedores.
