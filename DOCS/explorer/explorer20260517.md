# Explorer Report — Internacionalização (Inglês como língua padrão)

**Data:** 2026-05-17  
**Solicitação:** Migrar o app para inglês como língua padrão; avaliar detecção automática do idioma do usuário.

---

## Diagnóstico atual

O app está 100% em português hardcoded. Não existe nenhuma infraestrutura de i18n (sem i18next, sem arquivos de tradução, sem constantes de string). As strings estão diretamente dentro dos componentes React e nos arquivos Rust.

**Volume estimado:** ~300–350 strings espalhadas em ~25 arquivos.

| Categoria | Arquivos mais afetados |
|---|---|
| UI labels / botões | `NotebookList`, `Sidebar`, `ChatSidebar`, `FormattingToolbar` |
| Mensagens de erro e status | `useChat.ts`, `SettingsModal`, `SaveToNoteModal` |
| System prompts da IA | `App.tsx` (bloco de ~77 linhas) |
| Definições dos `/comandos` | `src/lib/commands.ts` + `commandParser.ts` |
| Backend Rust | `src-tauri/src/lib.rs`, `documents.rs` |

---

## Detecção de idioma do usuário

**Sim, é totalmente viável.** O Tauri roda uma WebView real, então `navigator.language` retorna o idioma configurado no sistema operacional do usuário — exatamente como um browser faria.

```typescript
// Funciona no Tauri sem nenhuma dependência extra
const userLang = navigator.language; // ex: "pt-BR", "en-US", "es-ES"
```

Isso permite:
- Servir inglês por padrão
- Detectar PT-BR e perguntar ao usuário se quer mudar para português
- Ou alternar automaticamente sem perguntar

A preferência pode ser salva no SQLite local (já existe infraestrutura de settings).

---

## Análise das duas abordagens

### Opção A — Troca direta para inglês (sem framework i18n)
**Dificuldade: Média**

Substituir todas as strings por inglês direto nos componentes. Sem arquivos de tradução, sem framework. Português desaparece do código.

- **Prós:** Simples, sem nova dependência, fácil de manter para um time pequeno
- **Contras:** Para adicionar PT-BR depois, precisa extrair tudo de novo
- **Adequada para:** "Inglês como língua única por agora, PT-BR talvez no futuro distante"

### Opção B — i18n com react-i18next + detecção automática
**Dificuldade: Média-Alta**

Instalar `i18next` + `react-i18next`, extrair todas as strings para arquivos JSON (`en.json`, `pt.json`), e usar `navigator.language` para detecção automática.

- **Prós:** Suporte multilíngue real, PT-BR pode ser mantido em paralelo, fácil adicionar novos idiomas no futuro
- **Contras:** ~2–3x mais trabalho inicial, nova dependência, overhead de manutenção
- **Adequada para:** "Quero inglês agora e PT-BR também disponível para usuários brasileiros"

---

## Questão dos `/comandos`

Os slash commands são um caso especial. O nome do comando (ex: `/pesquisa`, `/resumir`) faz parte da experiência de escrita — o usuário digita no editor.

Duas opções:
1. **Traduzir para inglês** (`/search`, `/summarize`, `/define`, etc.) — limpo, cosmopolita
2. **Aliases bilíngues** — `/pesquisa` e `/search` funcionam ao mesmo tempo

Como o projeto ainda não tem usuários ativos, a opção 1 (inglês puro) é recomendada — sem bagagem de compatibilidade.

---

## Recomendação

**Para este momento do projeto:** Opção A (troca direta para inglês) + detecção via `navigator.language` para guardar preferência futura.

Razão: o app está em fase pré-lançamento sem usuários ativos, a Opção A entrega o objetivo imediato com menos complexidade. Se o PT-BR for necessário mais tarde (ex: versão localizada para Brasil), extrai-se as strings para um framework naquele momento — mas não vale o overhead agora.

**O que fazer com os system prompts da IA:** Os prompts em `App.tsx` instruem a IA a responder em português. Isso precisa mudar para inglês também — e idealmente para responder no idioma detectado do usuário, o que melhora muito a experiência para qualquer língua.

## Tela de primeiro acesso (Onboarding)

**Dificuldade: Baixa**

Exibida apenas uma vez, na primeira abertura do app. Após o usuário completar o setup ou fechar, a tela nunca mais aparece (flag salva no SQLite local).

### Lógica de disparo

```typescript
// Na inicialização do app, checar se já passou pelo onboarding
const onboardingDone = await getOnboardingFlag() // lê do SQLite
if (!onboardingDone) showOnboarding()
```
