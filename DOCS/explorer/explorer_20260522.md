# Relatório de Análise de Viabilidade — 7 Tarefas
**Data:** 2026-05-22  
**Status:** Exploração completa

---

## Sumário Executivo

Todas as 7 tarefas são **viáveis**. A maioria (5/7) possui **dificuldade baixa/média**, mas 2 tarefas (mesclar notas e linkar notas) requerem atenção arquitetural. O projeto está bem estruturado para suportar essas implementações.

---

## Contexto do Projeto

### Stack Atual
- **Frontend:** React 18 + TypeScript + Tiptap 2 (WYSIWYG editor)
- **Backend:** Tauri 2 (Rust) + SQLite (monet.db)
- **Persistência:** src/storage/ (abstraction layer)
- **Model de dados:** Notebook, Note (title, content markdown, tags), AiResponse, Document

### Estado Atual
- Editor migrado de CodeMirror 6 para Tiptap 2
- Blocos de embed (embed-block) já implementados com toggle colapsável
- Sistema de /comandos totalmente funcional
- searchNotes() já existe (busca título + content + tags)
- RAG por caderno com sqlite-vec

## Análise das 7 Tarefas

### 1. Toggle Inline — Bloco Colapsável Inline no Editor

**Descrição:** Incluir bloco de toggle que pode ser expandido/colapsado inline.

**Viabilidade:** VIÁVEL — Dificuldade: BAIXA

**Por que?**
- Blocos de embed (embed-block) já existem em Tiptap e funcionam como toggles
- A UI está implementada em EmbedBlockView.tsx
- Falta apenas expor para o usuário criar toggles inline

**Arquivos Impactados:**
- src/components/Editor/EmbedBlock.ts
- src/components/Editor/EmbedBlockView.tsx
- src/components/Editor/CommandAutocomplete.tsx
- src/components/Editor/Editor.tsx

**Recomendações:**
1. Reutilizar EmbedBlockView (está pronto)
2. Adicionar variante do EmbedBlock onde commandId é nulo
3. Alternativa: /toggle command que insere bloco vazio

**Estimativa:** 2-4 horas

---

### 2. Abrir Primeira Nota ao Clicar no Caderno

**Descrição:** Ao clicar em um caderno, abrir automaticamente a primeira nota.

**Viabilidade:** VIÁVEL — Dificuldade: BAIXA

**Por que?**
- Arquitetura já suporta
- NotebookList clica e chama onSelect(notebookId)
- Falta: automaticamente selecionar primeira nota após setActiveNotebookId()

**Arquivos Impactados:**
- src/App.tsx (adicionar lógica após setActiveNotebookId)

**Estimativa:** 30 minutos

---

### 3. Mesclar Notas — Juntar Conteúdo de 2+ Notas

**Descrição:** Selecionar múltiplas notas e mesclar conteúdo em uma só.

**Viabilidade:** VIÁVEL — Dificuldade: MÉDIA

**Desafios:**
1. Seleção múltipla: Sidebar não suporta. Precisa state selectedNoteIds
2. Ordem de merge: cronológica? alfabética?
3. Título resultante: qual escolher?
4. Responses IA: copiar para nota mesclada? deletar?
5. Embed blocks: preservar markdown?

**Arquivos Impactados:**
- src/App.tsx (estado selectedNoteIds)
- src/components/Sidebar/Sidebar.tsx (checkboxes)
- src/storage/tauri.ts (mergeNotes method)
- src/storage/browser.ts
- src/storage/index.ts (interface)

**Recomendações (MVP):**
- Seleção de 2 notas via dropdown/modal
- Append conteúdo: A.content = A.content + "--- MERGED FROM:" + B.title + B.content
- Deletar B, preservar responses de A

**Estimativa:** 4-6 horas (MVP); +2-3 com seleção múltipla visual

### 4. Busca Avançada — Dentro da Nota Aberta (Ctrl+F)

**Descrição:** Busca de texto dentro do conteúdo da nota atual.

**Viabilidade:** VIÁVEL — Dificuldade: BAIXA

**Por que?**
- Tiptap expõe getJSON() e getText()
- Browser nativo Ctrl+F já funciona
- Customizado é trivial em React

**Implementação:**
1. Componente SearchInNote
2. Input + highlighting
3. Navegação: "1/5 matches"

**Arquivos Impactados:**
- src/components/Editor/SearchInNote.tsx (novo)
- src/components/Editor/Editor.tsx
- src/lib/ (helper)

**Recomendações:**
- Usar Ctrl+F nativo (já funciona)
- Se customizado: modal simples com regex

**Estimativa:** 1-2 horas

---

### 5. Linkar Notas — Criar Links Entre Notas

**Descrição:** Hyperlinks que apontam para outras notas dentro do app.

**Viabilidade:** VIÁVEL — Dificuldade: MÉDIA

**Desafios:**
1. Formato do link: monet://note/[id] ou note:[id]?
2. Link quebrado: se nota deletada, link fica morto
3. Navegação: clicar abre a nota (handler em Editor)
4. Busca de nota para linkar: modal similar a /docs

**Arquivos Impactados:**
- src/App.tsx (deep-link handler)
- src/components/Editor/Editor.tsx (click handler)
- src/types/index.ts (opcional NoteLink)

**Recomendações (MVP):**
- Markdown link simples: [text](monet://note/[id])
- Handler de click no Editor
- Modal de busca reutilizando searchNotes()

**Estimativa:** 3-4 horas (MVP); +2-3 com UI visual customizada

### 6. Espaço em Branco Final — Padding Inferior na Nota

**Descrição:** Garantir espaço em branco final para texto não ficar colado na borda.

**Viabilidade:** VIÁVEL — Dificuldade: BAIXA

**Por que?**
- Puramente CSS/layout
- Tiptap permite customização

**Soluções:**
1. CSS: adicionar padding-bottom: 400px ou 50vh ao .ProseMirror
2. Extensão Tiptap: parágrafo vazio final
3. Dinâmico: contar linhas

**Arquivos Impactados:**
- src/components/Editor/Editor.module.css

**Solução Recomendada:**
```css
.editor :global(.ProseMirror) {
  padding-bottom: 50vh;
}
```

**Estimativa:** 15 minutos

### 7. Busca Avançada Global (Ctrl+P)

**Descrição:** Command palette que busca em conteúdo de todas as notas.

**Viabilidade:** VIÁVEL — Dificuldade: MÉDIA

**Por que?**
- searchNotes() já existe (título + content + tags)
- Modal/palette trivial em React
- Ctrl+P hotkey simples

**Implementação:**
1. Hotkey listener: Ctrl+P abre modal
2. Input de busca
3. storage.searchNotes(query)
4. Dropdown com resultados (título + preview)
5. Enter abre nota

**Desafio: Relevância**
searchNotes() usa LIKE simples. Para ranquear:
- Matches em title > content
- Matches no início > meio
- Pode fazer ranking em JS pós-busca

Alternativa: FTS5 depois se performance crítica (requer migração schema).

**Arquivos Impactados:**
- src/components/SearchPalette/SearchPalette.tsx (novo)
- src/App.tsx (hotkey, estado)
- src/storage/tauri.ts (considerar otimizar searchNotes)

**Recomendações:**
1. Reutilizar modal existente
2. searchNotes() como está + ranking JS
3. FTS pode aguardar depois

**Estimativa:** 2-3 horas

## Recomendações Finais

### Arquitetura
- Projeto bem estruturado para essas mudanças
- Abstraction layer (storage/) permite mudanças sem quebrar frontend
- Tiptap oferece extensões prontas

### Riscos Mitigáveis
- **Responses IA órfãs (merge):** Decidir política antes
- **Relevância de busca:** MVP com ranking simples, otimizar depois
- **Links quebrados:** Validar ao carregar nota

## Conclusão

**Todas as 7 tarefas são viáveis.** Maioria (5/7) implementável em poucas horas. As 2 mais complexas (mesclar, linkar) requerem atenção arquitetural, mas sem bloqueadores técnicos. Projeto em boa posição para absorver essas mudanças.

Recomenda-se começar pelas tarefas baixa dificuldade (#2, #6) para ganhar momentum, depois médias (#4, #7, #1) e complexas (#3, #5).
