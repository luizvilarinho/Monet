# MSIX packaging (Microsoft Store)

Tauri only produces `.exe`/`.msi`. The Microsoft Store needs an `.msix`. These
scripts wrap the built `monet.exe` into an MSIX using the Windows SDK.

**Why MSIX:** when submitted to the Store, Microsoft signs the package for free
and hosts + auto-updates it. No paid Authenticode certificate is needed (the
`.exe`/`.msi` route would require one).

## Prerequisites

1. Windows 10/11 SDK installed (provides `makeappx.exe` + `signtool.exe`).
2. App built: `npm run tauri:build` (produces `src-tauri/target/release/monet.exe`).
3. The visual assets in `src-tauri/icons/` (`Square*Logo.png`, `StoreLogo.png`),
   generated by `npm run tauri icon`.

## Build for the Store (real submission)

Run **without** `-Sign`. Use the exact identity values from Partner Center
(your app > Product management > **Product identity**):

```powershell
./build-msix.ps1 `
  -IdentityName        "<Package/Identity/Name>" `
  -Publisher           "<Package/Identity/Publisher, e.g. CN=...>" `
  -PublisherDisplayName "<Package/Properties/PublisherDisplayName>"
```

Output: `src-tauri/target/msix/MonetNotes_<version>_x64.msix`. Upload that file
in Partner Center — the Store re-signs it. Do NOT self-sign Store packages.

## Build for local testing (sideload)

Run **with** `-Sign`. This creates a throwaway self-signed cert (local only):

```powershell
./build-msix.ps1 -Sign
```

Then, in an **Administrator** PowerShell, trust the cert and install:

```powershell
Import-Certificate -FilePath ".\src-tauri\target\msix\MonetNotesTest.cer" -CertStoreLocation Cert:\LocalMachine\TrustedPeople
Add-AppxPackage -Path ".\src-tauri\target\msix\MonetNotes_<version>_x64.msix"
```

## Notes

- The version comes from `tauri.conf.json` and is forced to 4 parts with the
  revision = 0 (Store requirement, e.g. `1.3.11.0`).
- The package is full-trust Win32 (`Windows.FullTrustApplication` +
  `runFullTrust`), the correct mode for a Tauri app.
- Inside the MSIX sandbox the GitHub-Releases auto-updater won't apply updates;
  Store builds update through the Store instead. Keep the GitHub build (with the
  updater) as the direct-download channel.
- WebView2 runtime is relied upon from the OS (preinstalled on Windows 11).
